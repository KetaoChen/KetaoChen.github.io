NgModule

The Angular app have at least one Module.
The Module can have many components.

Angular Component

Angular app is composed of components.
In each component, there basically four documents.

  1. .html: the frame of the website
  2. .css: the view of the website
  3. .ts: the frontend logic, which controls the work flow
  4. .spec: the test document

Work Flow

  1. Start from app.component
  2. The router is responsible for the navigation of the website.
  3. In the view, when an event happens, the logic is written in the component.ts document.

Tech

Data Binding:
a) Interpolation: one way data binding. Take data from a component property and show it in the view.
b) Binding an event on the view to a method in the component.ts.
c) NgModel: two way binding, banana in a box. Changed the property of that component.

Dependency Injection:
In Angular DI is a built in feature.
Inject the services into components.

Session Storage:
Can store a session by setting items.
