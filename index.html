NgModule

The Angular app have at least one Module.
The Module can have many components.

Angular Component

Angular app is composed of components.
In each component, there basically four documents.

  1. .html: the frame of the website
  2. .css: the view of the website
  3. .ts: the frontend logic, which controls the work flow
  4. .spec: the test document

Work Flow

  1. Start from app.component
  2. The router is responsible for the navigation of the website.
  3. In the view, when an event happens, the logic is written in the component.ts document.

Tech

Data Binding:
a) Interpolation: one way data binding. Take data from a component property and show it in the view.
b) Binding an event on the view to a method in the component.ts.
c) NgModel: two way binding, banana in a box. Changed the property of that component.

Dependency Injection:
In Angular DI is a built in feature.
Inject the services into components.

Session Storage:
Can store a session by setting items.

SpringBoot

    • Spring Boot
      • MVC
      • Auto Configure
      • DI & Inverse of Control

    • JWT (Json Web Token)

What is Web service?

a) Design for machine-to-machine(application-to-application).
b) Interoperable. (platform independent)
c) Communication over a network.

RESTful

a) Make best use of HTTP: hyper text transfer protocol. to manipulate resource.
b) HTTP request methods: GET, POST, PUT, DELETE. (CRUD)
c) HTTP status code: 200, 301, 404
d) Resource. URI(uniform resource Identifier) Assign a URI for each resource.

Spring

MVC (Model-View-Controller):

  • The model is responsible for manipulate the data of the application.
  • The view is the presentation of the model in a particular format.
  • The controller interacts with different models according to differnt inputs.
  • Make application loosely coupled.

Auto Configure:
a) Dispatcher servlet: front controller.
b) Mapping the URI to corresponding method.

DI & Inverse of Control:
@AutoWire: Control all the beans to inject object to achieve inversion of control.

JPA(Java Persistence API):
Hibernate: Provides a Entity (Object) relationship mapping between the classes and tables .

JWT (Json Web Token)

  1. No expiration.
  2. Stateles.
